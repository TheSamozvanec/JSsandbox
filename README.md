Очень простое решение, для небольштх эксперементов на JS прямо в телефоне
Проект содержит всего 3 файла index.html, style.css и script.js
Их нужно скоприровать в 1 папку и запустить в браузере index.html
В телефоне работает в приложении "файловый менеджер +" 
![icon](./images/app_icon.jpg)
Открывшаяся страница будет выглядеть примерно так:
![page](./images/page.jpg)
Сверху кнопки load, save, local, run, clear и snip
Load - загрузить текст. Можно выбрать файл и он зпгрузится в верхнее окно приложения в виде текста Предполагается что это будет текст скрипта.
Save - текст скрипта будет сохранен. К сожалению сохраняется только в загрузки. 
Local - текст программы сохранится в localStorage и откроется при перезагрузке страницы или открытии страницы
RUN - запуск скрипта в верхнем окне
clear - очистить верхнее окно
snip - коллекция сниппетов, которую я еще не дописал, но кое что уже есть
Поле с именем файла и еще 2 кнопки, позволяющие двигать курсор в верхнем окне (поле ввода скрипта)
курсор в верхнем окне кнопками двигать бывает удобно, когда сенсор на телефоне не слушается (у меня по краям экрана прям бесит иногда)
Далее верхнее окно - поле для ввода скрипта, который вы написали (загрузили) и хотите запустить 
Самое нижнее окно - monitor, там отображается служебная и справочная информация во время написания скриптов (предварительная проверка скрипта, описание снипета и т.п.), ошибки при выполнении скрипта и это окно можно использовать для вывода данных в своем скрипте.
При написании скрипта доступны внутренние методы print() для отображения данных, cls() для очистки монитора (нижнего окна) и printObj() для распаковки в мониторе верхнего уровня объекта
Пример использования printObj()
let obj={a:10,b12, c:14};
printObj(obj);
При нажатии RUN вы увидите распаковку объекта в нижнем окне.
printObj не ломает html монитора, если вы в своем скрипте что-то туда закинули (кнопку или див), так как использует innerHTML
print(Obj) отобразит [object Object], но им удобней мониторить переменные. print использует textContent из-за этого он ломает содержимое html монитора
Методы print и printObj я оставил в альтернативу console.log так как консоль использовать в телефоне неудобно, а если не подобрать браузер и вовсе не возможно.
Также доступны объекты monitor и style. Монитор - это див в конце html самой страницы, стиль - <style> в <head> страницы
В монитор можно писать html прям в вашем скрипте monitor.innerHTML= `<button class='key'>Нажми</button>` для удобства есть сниппет (даже группа снипетов) 
В стиль можно закидывать css также прям innerHTML и конечно же есть группа сниппетов

И про настройки: 
При загрузке приложения, код проверяет localStorage и если вы раньше не сохраняли настройки - установит default, который прописан в начале скрипта приложения (script.js)
В сниппетах есть удобные кнопки настройки
local - загрузить настройки из localStorage
save - сохранить текущие настройки в localStorage
default- сбросить настройки
delete - удалить настройки из localStorage
Help - справка по настройкам (выйдет в мониторе) 
setup в поле ввода появится метод передачи объекта настроек и сам объект, можно подправить его и нажать run. Установки будут работать до березагрузки, если их не сохранить (сниппет в настройках save)
start - после вывода очередного снипета курсор встает в его начало
end - после вывода очередного снипета курсор встает в его конец
select - сниппет выделяется
index - курсор встает в точку сниппета, указанную в самом сниппете
autosave - текст автоматически сохраняется в localStorage сразу после нажатия на RUN
