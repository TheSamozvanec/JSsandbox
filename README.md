Очень простое решение, для небольштх эксперементов на JS прямо в телефоне.<br>
Проект содержит всего 3 файла index.html, style.css и script.js <br>
Их нужно скоприровать в 1 папку и запустить в браузере index.html <br>
В телефоне работает в приложении "файловый менеджер +" <br>
![icon](./images/app_icon.jpg) <br>
<br>
В другом файловом менеджере запускается только html а стилизация и скрипт игнорируются. <br>
<br>
Открывшаяся страница будет выглядеть примерно так: <br>
![page](./images/page.jpg) <br>
<br>
Сверху кнопки load, save, local, run, clear и snip <br>
Load - загрузить текст. Можно выбрать файл и он зпгрузится в верхнее окно приложения в виде текста Предполагается что это будет текст скрипта. <br>
Save - текст скрипта будет сохранен. К сожалению сохраняется только в загрузки. <br>
Local - текст программы сохранится в localStorage и откроется при перезагрузке страницы или открытии страницы <br>
RUN - запуск скрипта в верхнем окне <br>
clear - очистить верхнее окно <br>
snip - коллекция сниппетов, которую я еще не дописал, но кое что уже есть <br>
Поле с именем файла и еще 2 кнопки, позволяющие двигать курсор в верхнем окне (поле ввода скрипта) <br>
курсор в верхнем окне кнопками двигать бывает удобно, когда сенсор на телефоне не слушается (у меня по краям экрана прям бесит иногда) <br>
Далее верхнее окно - поле для ввода скрипта, который вы написали (загрузили) и хотите запустить  <br>
Самое нижнее окно - monitor, там отображается служебная и справочная информация во время написания скриптов (предварительная проверка скрипта, описание снипета и т.п.), ошибки при выполнении скрипта и это окно можно использовать для вывода данных в своем скрипте. <br>
При написании скрипта доступны внутренние методы print() для отображения данных, cls() для очистки монитора (нижнего окна) и printObj() для распаковки в мониторе верхнего уровня объекта <br>
Пример использования printObj() <br>
let obj={a:10,b12, c:14}; <br>
printObj(obj); <br>
При нажатии RUN вы увидите распаковку объекта в нижнем окне. <br>
printObj не ломает html монитора, если вы в своем скрипте что-то туда закинули (кнопку или див), так как использует innerHTML <br>
print(Obj) отобразит [object Object], но им удобней мониторить переменные. print использует textContent из-за этого он ломает содержимое html монитора <br>
Методы print и printObj я оставил в альтернативу console.log так как консоль использовать в телефоне неудобно, а если не подобрать браузер и вовсе не возможно. <br>
Также доступны объекты monitor и style. Монитор - это див в конце html самой страницы, стиль -  тег style в теге head страницы <br>
В монитор можно писать html прям в вашем скрипте monitor.innerHTML= `<button class='key'>Нажми</button>` для удобства есть сниппет (даже группа снипетов) <br>
В стиль можно закидывать css также прям innerHTML и конечно же есть группа сниппетов <br>

И про настройки: <br> 
При загрузке приложения, код проверяет localStorage и если вы раньше не сохраняли настройки - установит default, который прописан в начале скрипта приложения (script.js) <br>
В сниппетах есть удобные кнопки настройки <br>
local - загрузить настройки из localStorage <br>
save - сохранить текущие настройки в localStorage <br>
default- сбросить настройки <br>
delete - удалить настройки из localStorage <br>
Help - справка по настройкам (выйдет в мониторе) <br>
setup в поле ввода появится метод передачи объекта настроек и сам объект, можно подправить его и нажать run. Установки будут работать до березагрузки, если их не сохранить (сниппет в настройках save) <br>
start - после вывода очередного снипета курсор встает в его начало <br>
end - после вывода очередного снипета курсор встает в его конец <br>
select - сниппет выделяется <br>
index - курсор встает в точку сниппета, указанную в самом сниппете <br>
autosave - текст автоматически сохраняется в localStorage сразу после нажатия на RUN <br>
<br>
Цвета кнопок сниппетов:<br> 
серый - просто сниппет;<br>
зеленый - многострочный синтаксис;<br>
синий - сложный сниппет с переходом на следующую подгруппу. Например Math.=>sin;<br>
красный - сниппет/кнопка с переходом на предыдущую группу сниппетов. Если выбрать сниппет после Math, меню сниппетов вернется в первое окно, а если "..." - будет возврат без вывода сниппета (такое есть в группах);<br>
Желтый - группа сниппетов (например HTML);<br>
